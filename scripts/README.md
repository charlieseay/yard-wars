# Birdies and Bourbon - Test Automation

Automated testing suite for on-device validation.

## Quick Start

**One command to rule them all:**

```bash
cd ~/Projects/birdies-and-bourbon
./scripts/test-automation.sh
```

This will:
1. ✅ Validate TypeScript syntax
2. ✅ Check dependencies
3. 🚀 Start dev server with monitoring
4. 📱 Wait for device connection
5. 🔍 Auto-run health checks on device
6. 📊 Generate test report when you Ctrl+C

## Individual Scripts

### Pre-flight Validation

Run before starting server to catch issues early:

```bash
./scripts/pre-flight.sh
```

Checks:
- Node/npm versions
- TypeScript syntax
- Required dependencies
- Expo CLI availability

### Start Test Server

Start dev server with automatic health monitoring:

```bash
./scripts/start-test-server.sh
```

Monitors:
- Device connections
- Bundle load events
- Runtime errors

Logs saved to `.test-data/logs/`

### Generate Test Report

After testing, generate a consolidated report:

```bash
./scripts/generate-test-report.sh
```

Report includes:
- Health check results (File API, SQLite, storage)
- Device connection log
- Error summary
- Recommendations

Reports saved to `.test-data/reports/`

## Health Check System

The app automatically runs a health check on device load. Results appear in:
- **Console logs** (visible in Expo dev tools)
- **Device storage** (`health-check-result.json`)
- **Test report** (generated by scripts)

### What It Checks

1. **File System API** - `expo-file-system` availability
2. **SQLite API** - `expo-sqlite` availability
3. **Storage Write** - Can write JSON to device
4. **Storage Read** - Can read back written data

### Reading Results

**On device (Expo dev tools):**
```

===================================

Checks:
  File System API: ✅
  SQLite API: ✅
  Storage Write: ✅
  Storage Read: ✅
```

```bash

```

## Workflow

### Automated Testing (Recommended)

```bash
# Run full suite
./scripts/test-automation.sh

# Scan QR code on iPhone
# App loads → health check auto-runs
# Play with the app
# Press Ctrl+C when done

```

```bash
# 1. Pre-flight
./scripts/pre-flight.sh

# 2. Start server
./scripts/start-test-server.sh

# 3. Test on device
# ... (manual testing)

# 4. Generate report (in another terminal)
./scripts/generate-test-report.sh
```

## Troubleshooting

### No device connection

1. Verify iPhone and Mac are on same WiFi
2. Check firewall isn't blocking port 8081
3. Update Expo Go app from App Store
4. Try clearing cache: `npx expo start -c`

### Health checks fail

1. Check Expo SDK version (current: 57)
2. Verify device permissions (Settings → Expo Go → Storage)
3. Try reinstalling Expo Go
4. Check for iOS compatibility issues

### TypeScript errors in pre-flight

```bash
# Run TypeScript check with details
npx tsc --noEmit

# Fix errors in reported files
# Re-run pre-flight
./scripts/pre-flight.sh
```
